# fourpack-loop — 让 AI 自己写、自己挑错、自己改

配合 fourpack 搭好的底子用。前提：当前文件夹里已经有 `roles.md`（PM / UI-UX / viewer 三个把关人）和四件套。没有就先跑 `fourpack` 把底子搭出来。

核心一句话：**写的人和检查的人分开**——别让同一个 AI 自己写完又自己说"完美"。这个技能把"写"和"三个角色挑错"拆开，循环到没硬伤为止。

## 怎么做（按顺序，每一步都说出来给人看）

### 1. 先对齐（最多问一两句）

用户给了功能需求（比如"现在页面空空的，给页面加上用户系统的组件和酷炫的动画"）。如果有真会影响做法的歧义，先 grill 一两句问清楚（比如"用户系统要不要真能注册登录，还是先做个样子？""动画是进场就够，还是要交互时也动？"）。问明白就停，别盘问。需求已经清楚就直接下一步。

### 2. maker 写第一版

按需求做出第一版。做的时候**简单说一下你打算怎么做、为什么这么做**（一两句），别闷头写完甩一坨。

### 3. 拉三个角色当独立检查官，逐个出声点评

读 `roles.md`，让三个角色**各自单独**挑刺。关键：现在你的身份是"检查官"，不是"作者"——**专门找毛病，别夸自己**。一个一个来，每个角色都把话说出来（这一段就是给观众看的"思考过程"）：

- **PM**：这功能到底帮用户解决了啥？做完怎么知道有用？有没有跑偏、加了用户没要的东西？
- **UI/UX**：第一次打开的人看不看得懂、知不知道下一步点哪？这些"酷炫动画"是真帮忙，还是花里胡哨、晃眼、挡住正事？
- **viewer**：扮一个挑剔、脾气还有点急的第一次用户，30 秒扫一眼说真话——"这我看懂了吗 / 是不是在装 / 加载或动画卡不卡 / 我还想不想再用"。

把三个角色挑出来的问题**列成一张清单**（哪个角色、挑出啥、严不严重）。

### 4. maker 按清单改

回到"作者"身份，挑真问题改。**说一下这轮改了啥、为什么**。一时兴起想加的、PM 说跑偏的，砍掉。

### 5. 再检查一轮

让三个角色再扫一遍：上一轮的问题修了没？改的时候有没有带出新毛病？没硬伤了就停；还有就回第 4 步。一般一到两轮就收敛，别无限纠结质感。

### 6. 收尾

做完了，用大白话告诉用户：这轮加了啥、三个角色挑过哪些、最后改成什么样。然后把这次进度顺手记一句进 `progress.md`（活不丢）。

## 红线（说人话 + 别自我放水）

- **写的人 ≠ 检查的人**：第 3 步必须真的换成"找毛病"的心态，不许走过场夸自己。这是整个循环的意义。
- **每一步都出声**：打算怎么做、每个角色挑了啥、改了啥——都说出来。别只甩最终结果（也方便录屏展示思考过程）。
- **零黑话**：跟用户讲的话里别甩 maker / checker / loop 这些词，就说"我先写、再换个身份挑错、再改"。
- **别过度**：动画/功能够用就行，PM 觉得没用的别硬塞；质感类小问题（不影响能不能用）点一下就行，别为它无限循环。
