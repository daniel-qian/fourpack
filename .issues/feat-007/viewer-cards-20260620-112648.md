## 最该改的 3 条（按对小白杀伤力排序）

### 1.【最致命】开场第一句把 "harness" 生甩，破了"说人话"的承诺，3 秒内劝退
- **位置**：第 2 节开场口播第一句 + 第 6 节连读稿第一段 + 第 1 节定位句（"提示词/上下文/harness/loop"）。
- **问题**：全片唯一卖点是"用大白话"，但开场第一句就有一个未翻译的英文工程词 `harness`。Venessa 第一句就卡壳出戏；这正是 roles.md 里 Fred/Venessa 反复点的"视频—产品气氛断裂"和 dogfood 红线（反黑话的工具自己一身黑话）。四个词连发本身也信息过载。
- **怎么改**：开场**第一句别报菜名**。要么四个词全部当场配一个三字大白话标签（"提示词=帮你问清楚 / 上下文=治失忆 / harness=活不丢 / loop=自己挑错"），要么**开场只抛痛点和'5 分钟懂+上手'，把四个词留到画面里的气泡**，等正片各段落地时再逐个点名。无论如何 harness 第一次出现必须当场翻译，不许裸奔。

### 2.【护城河没演出来】B6 "三个角色挑错"全靠口播+字幕解释，不是"可视化"，且观众会当成"AI 自己跟自己聊"
- **位置**：第 4 节 B6（"可视化思考 · 给足特写"）+ 第 5 节收尾。
- **问题**：脚本自己在最前面（第 12–18 行）就警告"loop 段被剪成'它又写了点东西'=白讲"，但 B6 的实际设计就是文字滚屏 + 口播替它解释。小白（Venessa）看到的是"一堆字在滚"，且会怀疑"这不就是 AI 自己跟自己说话换了个名字"——**"写的人 ≠ 检查的人"这个唯一的 punchline 没有在画面上被立起来**。
- **怎么改**：给三个角色**做视觉区分**而不是纯滚字——三张角色卡/三个头像/三种颜色气泡，让"换身份"在画面上可见；至少给一处"角色挑出毛病 → maker 当场认错改掉"的**前后对比特写**（改之前 vs 改之后并排），把"它真把自己的东西推翻了"这件事拍成肉眼可见的动作，而不是字幕陈述。这是把抽象 loop 变成可看动作的唯一办法。

### 3.【赌博】护城河镜头尚未录制，且能否录出取决于平台是否配合演 roleplay
- **位置**：第 8 节待定项 1（"B5–B7 要补录"）+ B6 对 `fourpack-loop` 技能的依赖。
- **问题**：全片最值钱的 2 分钟此刻**还不存在**，且 `fourpack-loop/SKILL.md` 里"每一步都出声、给观众看思考过程"是**对 AI 的请求**，不是平台保证——minimax code 当时可能把 roleplay 折叠、可能一句话带过、可能根本不分角色出声。押全片护城河在一段不可控镜头上，风险极高。
- **怎么改**：**先录一条 B5–B7 的样片验证**——确认 minimax code 真能把三个角色逐个出声挑错的过程显式展示出来，再定剪辑方案。如果平台不配合，需要预案 B：手动把三轮（写→PM/UI/viewer 挑错→改）做成**分镜重演/字幕卡复刻**，别指望一镜到底真机录。技能文件能描述流程，但不能保证录屏画面好看——这两件事脚本现在混为一谈了。

## 总判

**不能直接拿去录续段/配音，要先改 2 处再录。**

- 开场口播（第 1 条）必须先改：`harness` 裸奔 + 四词连发会在前 3 秒劝退正是目标人群的小白，这是配音脚本层面的硬伤，改字就行，成本低、收益最大。
- B6 护城河的**呈现方式**（第 2 条）必须在录之前先定成"视觉分角色 + 前后对比",否则按现稿录出来就是脚本自己警告的"它又写了点东西",白录。
- 第 3 条是录制风险，要先打样片，但不属于"改脚本文字"——属于"录之前先验证平台"。

前半段（B0–B4）口播扎实、说人话基本达标、self-demo 收尾有力，**那部分可以直接用**。问题全集中在新增的开场第一句和 loop 续段——也正好是这版改动最重的地方。
